## 1. 背景与目标

### 1.1 双体系问题描述

当前项目暗色模式存在两套独立的 CSS 选择器体系，导致生产环境出现暗色覆盖盲区：

**主体系（102 个文件）**：使用 `body.dark .xxx` 选择器
- 通过 ThemeContext 的 `document.body.classList.toggle('dark')` 机制切换
- 生产环境完全生效
- 已覆盖全局 CSS 变量定义及绝大多数业务页面

**遗留体系（18 个文件）**：使用 `[data-theme='dark'] .xxx` 选择器
- 依赖 HTML 元素的 `data-theme` 属性切换
- **生产环境完全失效**（ThemeContext 不设置该属性）
- 涉及多个核心业务页面的暗色样式覆盖盲区

### 1.2 统一目标

- **消除覆盖盲区**：确保所有业务页面暗色模式样式完整生效
- **提升暗色模式完整性**：全站暗色体验一致，无白底穿透、色彩错位
- **简化维护成本**：未来新增页面只需遵循单一选择器规范
- **零回归风险**：浅色模式视觉效果保持不变

### 1.3 业务影响

本次统一涉及 18 个业务页面的暗色体验修复，覆盖用户核心使用场景：

- 社区互动类：评论区、评论项、嵌套评论
- 内容消费类：配方详情页、烹饪日志页、营养仪表板
- 工具功能类：厨房工具页、购物清单页、食谱对比页
- 个性化类：偏好页、筛选配置

## 2. 统一方案

### 2.1 推荐选择器

**推荐选择：`body.dark`**

选择理由：
- **覆盖范围最广**：102 个文件已采用此模式，占绝对多数
- **与实现一致**：ThemeContext 实际使用 `body.classList` 切换机制
- **全局变量完整**：`frontend/src/global.css` 已通过 `body.dark` 完整定义所有 CSS 变量
- **维护成本最低**：仅需修改 18 个遗留文件，无需修改主体系

### 2.2 否决方案说明

**否决：`[data-theme='dark']`**

否决理由：
- 仅 18 个文件使用，占比极小
- 与 ThemeContext 实现机制不一致，需要修改主题切换逻辑
- 全局 CSS 变量未基于此选择器定义，需大量变量迁移工作
- 会导致 102 个主体系文件需要反向修改，风险极高

### 2.3 约束规则

**禁止新增第三套选择器**
- 不允许引入 `html.dark`、`:root.dark` 或其他新的选择器形式
- 保持选择器体系单一化

**不修改 ThemeContext 实现**
- 保持现有主题切换逻辑不变
- 仅在 CSS 层进行选择器统一
- 不涉及 JS/TS 代码修改

### 3.1 通用替换规则

| 遗留选择器 | 统一选择器 | 说明 |
|-----------|-----------|------|
| `[data-theme='dark']` | `body.dark` | 所有遗留文件的顶层暗色选择器 |
| `[data-theme='dark'] .xxx` | `body.dark .xxx` | 带类名限定的选择器 |
| `html[data-theme='dark']` | `body.dark` | 带 html 前缀的选择器 |
| `:root[data-theme='dark']` | `body.dark` | 带 :root 前缀的选择器 |

## 7. 风险评估

### 7.1 低风险项

**主题切换瞬时无闪烁**
- 风险等级：低
- 现状：ThemeContext 已实现无闪烁切换
- 回归要求：验证切换过程无白屏、无闪烁
-  mitigation：切换前后保持 CSS 变量继承一致

**localStorage 主题持久化**
- 风险等级：低
- 现状：已实现本地存储
- 回归要求：刷新后主题保持
- mitigation：不涉及存储逻辑修改，仅 CSS 层变更

### 7.2 中风险项

**浅色专属变量被暗色规则覆盖**
- 风险等级：中
- 风险场景：18 个文件中可能存在同时定义浅色和暗色规则的情况
- 潜在问题：CSS 选择器优先级变化导致浅色样式被覆盖
- 检查方式：逐个文件检查选择器层级，确保浅色规则优先级正确
- mitigation：保持选择器结构不变，仅替换前缀，不调整顺序

**暗色变量继承层级完整性**
- 风险等级：中
- 风险场景：global.css → 页面 CSS 的变量继承链可能断裂
- 潜在问题：某些页面定义了独立的暗色变量，切换选择器后变量不生效
- 检查方式：对比每个页面的暗色变量定义，确保与 global.css 一致
- mitigation：仅替换选择器前缀，不动变量名及值

### 7.3 高风险项

本次变更无高风险项。所有变更均为 CSS 选择器文本替换，不涉及：
- 主题切换逻辑修改
- 全局变量定义修改
- 业务逻辑修改
- 数据结构修改

## 8. 设计原则

### 8.1 最小化变更原则

- **只改选择器，不动颜色值**
- 保持所有颜色定义、数值、单位完全不变
- 保持所有选择器的层级结构不变
- 仅替换选择器前缀字符串
- 不调整 CSS 规则顺序
- 不新增或删除任何 CSS 规则

### 8.2 一致性原则

- **与主体系 102 个文件的视觉风格完全一致**
- 暗色背景色：与主体系保持一致
- 暗色文字色：与主体系保持一致
- 暗色强调色：与主体系保持一致
- 交互态（hover/active/focus）：与主体系保持一致
- 组件样式（卡片、按钮、输入框）：与主体系保持一致

### 8.3 可维护性原则

- **未来新页面只需使用 `body.dark`，无需考虑其他选择器**
- 文档化统一选择器规范
- 在代码规范中明确暗色模式选择器要求
- Code Review 环节检查暗色选择器使用
- 不允许再引入 `[data-theme='dark']` 选择器

### 8.4 零回归原则

- **浅色模式必须与基线完全一致**
- 变更前后浅色模式截图对比 100% 匹配
- 所有交互状态在浅色模式下无变化
- 所有装饰元素（阴影、边框、分割线）无变化
