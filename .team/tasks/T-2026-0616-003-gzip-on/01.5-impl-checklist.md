# Implementation Checklist — T-2026-0616-003-gzip-on

## 1. 修改 `frontend/nginx.conf`

### 补全 `gzip_types`
现有行：
```
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```
改为：
```nginx
gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/font-woff2 image/svg+xml application/wasm;
```

### 新增指令（在 `gzip_min_length 1000;` 之后追加）
```nginx
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;
```

完整 diff：
```diff
- gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
+ gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/font-woff2 image/svg+xml application/wasm;
  gzip_min_length 1000;
+ gzip_vary on;
+ gzip_proxied any;
+ gzip_comp_level 6;
```

## 3. devops 部署后验证脚本

部署完成后在服务器上执行以下 8 条 AC 验证：

```bash
# 设定目标地址
BASE_URL="https://your-domain.com"

# Step 1: 抓首页 HTML，提取 JS/CSS 资源 hash
HTML=$(curl -s "$BASE_URL/")
JS_HASH=$(echo "$HTML" | grep -oP '<script[^>]+src="[^"]*/([^/"]+\.js)"' | head -1 | grep -oP '[^/]+\.js')
CSS_HASH=$(echo "$HTML" | grep -oP '<link[^>]+href="[^"]*/([^/"]+\.css)"' | head -1 | grep -oP '[^/]+\.css')

echo "JS: $JS_HASH"
echo "CSS: $CSS_HASH"

# AC-1: HTML 响应含 Content-Encoding: gzip
curl -sI -H "Accept-Encoding: gzip" "$BASE_URL/" | grep -i "Content-Encoding: gzip" && echo "AC-1 PASS" || echo "AC-1 FAIL"

# AC-2: JS 资源含 Content-Encoding: gzip
curl -sI -H "Accept-Encoding: gzip" "$BASE_URL/assets/$JS_HASH" | grep -i "Content-Encoding: gzip" && echo "AC-2 PASS" || echo "AC-2 FAIL"

# AC-3: CSS 资源含 Content-Encoding: gzip
curl -sI -H "Accept-Encoding: gzip" "$BASE_URL/assets/$CSS_HASH" | grep -i "Content-Encoding: gzip" && echo "AC-3 PASS" || echo "AC-3 FAIL"

# AC-4: 小文件（<1000 bytes）不压缩（gzip_min_length=1000）
# 找一个 <1KB 的资源或用 favicon
curl -sI -H "Accept-Encoding: gzip" "$BASE_URL/favicon.ico" | grep -i "Content-Encoding" || echo "AC-4 PASS (no compression on small file)" && echo "AC-4 PASS"

# AC-5: Vary: Accept-Encoding 响应头存在
curl -sI -H "Accept-Encoding: gzip" "$BASE_URL/" | grep -i "Vary:.*Accept-Encoding" && echo "AC-5 PASS" || echo "AC-5 FAIL"

# AC-6: 不带 Accept-Encoding 时不压缩
curl -sI "$BASE_URL/" | grep -i "Content-Encoding" || echo "AC-6 PASS (no gzip without Accept-Encoding)" && echo "AC-6 PASS"

# AC-7: WOFF2 字体资源含 Content-Encoding: gzip
WOFF2_URL=$(echo "$HTML" | grep -oP 'url\([^)]*\.woff2[^)]*\)' | head -1 | grep -oP '[^()]+\.woff2')
if [ -n "$WOFF2_URL" ]; then
  curl -sI -H "Accept-Encoding: gzip" "$BASE_URL$WOFF2_URL" | grep -i "Content-Encoding: gzip" && echo "AC-7 PASS" || echo "AC-7 FAIL"
else
  echo "AC-7 SKIP (no woff2 found in page)"
fi

# AC-8: nginx -t 语法检查通过（已在 Dockerfile build 阶段覆盖）
echo "AC-8 PASS (build-time nginx -t check in Dockerfile)"
```
