## 五、Bug 清单

### 🔴 P0 - 详情页崩溃（数据格式不一致）

**现象**：部分食谱详情页（如"法式马卡龙"）加载时 ErrorBoundary 捕获异常，显示"页面出错了"。

**错误信息**：
```
TypeError: Cannot read properties of undefined (reading 'match')
    at y (useSpeechSynthesis-jbR9ct9O.js:1:309)
```

**根因**：
- seed 数据中 `steps` 字段格式不一致
- **水煮鱼**（正常）：`steps` 是对象数组 `[{stepNumber, content}]`
- **法式马卡龙**（崩溃）：`steps` 是纯字符串数组 `["步骤1", "步骤2"]`
- 前端代码 `RecipeDetailPage.tsx` 假设 `steps` 始终是对象数组：
  - `step.stepNumber` → 字符串无此属性 → `undefined`
  - `step.content` → 字符串无此属性 → `undefined`
  - `speak(step.content)` → `speak(undefined)` → `new SpeechSynthesisUtterance(undefined)` → 内部调用 `.match()` 崩溃

**修复方案**：
1. **数据层**：统一 seed 数据中的 steps 格式，全部改为对象数组
2. **代码层**：在 `RecipeDetailPage.tsx` 中添加兼容处理：
   ```tsx
   const normalizedSteps = recipe.steps?.map((step, index) =>
     typeof step === 'string'
       ? { stepNumber: index + 1, content: step, image: null }
       : step
   ) ?? []
   ```
3. **防御层**：`useSpeechSynthesis.ts` 中 `speak()` 函数添加空值检查：
   ```tsx
   const speak = useCallback((text: string, onEnd?: () => void) => {
     if (!supported || !text) return
     // ...
   }, [supported])
   ```

**影响范围**：
- 所有 steps 为字符串数组的食谱详情页都无法访问
- 用户无法查看这些食谱，直接影响核心功能

---

### 🟡 P1 - 步骤图片严重裁切

**现象**：制作步骤中的图片被渲染为 200×130，但原图尺寸为 800×450/400×300/800×1198 等，图片被严重压缩裁切。

**根因**：`.step-image` 的 CSS 可能设置了固定宽度（200px），高度自适应导致比例失调。

**修复建议**：
- 增大步骤图片显示尺寸，或改为点击放大查看
- 设置 `object-fit: cover` + 固定宽高比容器
