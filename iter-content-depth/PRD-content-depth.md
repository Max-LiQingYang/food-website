## 1. 数据摸底

### 1.1 数据源与方法

**分析对象**：`backend/seeds/seed.js` 中 94 道食谱完整数据
**分析方法**：Node.js 脚本解析 seed.js 文本，提取每道食谱的 `description`、`tips`、`culturalBackground` 去空白后计数字符长度
**注**：因本地 MariaDB 未运行，分析基于 seed.js 静态文件，其结果应与生产 DB 完全一致（seed.js 即数据源）

### 1.2 描述（description）长度分布

| 区间 | 标准 | 数量 | 占比 |
|------|------|------|------|
| 短 | <80字 | 36 | 38.3% |

**统计**：最短 53 字（芒果糯米饭），最长 ~180 字，中位数约 82 字，平均值约 85 字

## 2. 内容增强方案（不依赖外部 AI API）

### 核心设计原则

- **零外部 API 依赖**：所有增强内容来自预编制的模板库和规则引擎，不调用任何 AI API
- **可复现**：seed.js 增强脚本可重复执行，生成结果一致
- **可维护**：模板按分类（cuisine × category）组织，易于增补和修正
- **渐进增强**：只增强需要增强的字段，不覆盖已有优质内容

### 2.1 P0：描述增强脚本 `enhance-descriptions.js`

**目标**：为描述 <100 字的食谱补充食材特色、口感、适合场景

#### 2.1.1 数据源设计

创建描述模板库 `data/description-templates.json`，按 `(cuisine, mainIngredient)` 两级索引：

```json
{
  "chinese": {
    "poultry": {
      "taste": ["鲜嫩多汁", "皮滑肉嫩", "酱香浓郁"],
      "texture": ["肉质紧实不柴", "入口即化", "弹牙爽口"],
      "scene": ["家庭聚餐首选", "下饭神器", "宴客硬菜", "便当必备"],
      "connectors": ["搭配", "佐以", "淋上", "蘸上"]
    },
    "seafood": {
      "taste": ["鲜美清甜", "原汁原味", "蒜香扑鼻"],
      "texture": ["肉质细嫩", "弹滑爽脆", "入口鲜美"],
      "scene": ["海鲜爱好者的心头好", "宴请佳品", "周末加餐"],
      "connectors": ["蒸制后", "出锅前", "浇上热油"]
    }
    // ... 更多主食材类型
  },
  "western": { /* ... */ },
  "japanese": { /* ... */ },
  "korean": { /* ... */ },
  "thai": { /* ... */ },
  // ...
}
```

#### 2.1.2 增强逻辑

```
FOR each recipe WHERE descLen < 100:
  1. 识别食谱的 cuisine（category 字段）和 mainIngredient（从 categoryTags 或 ingredients 推断）
  2. 从模板库随机（带种子保证可复现）选取 1-2 个口感词 + 1 个场景词
  3. 用连接词组装成 1-2 句补充描述
  4. 将增强内容追加到原 description 末尾（不替换原有内容）
  5. 确保增强后总字数 ≥ 100 字
```

| 食谱 | 原描述 | 增强后 |
|------|--------|--------|

| 麻婆豆腐（60字） | "嫩滑豆腐裹满香辣肉末酱汁..." | 补充：**"花椒的麻与辣椒的香在舌尖交错激荡，豆腐入口即化，肉末酥香入味。一碗麻婆豆腐配白米饭，是川渝人最朴素的幸福。"** |

### 2.2 P1：小贴士增强脚本 `enhance-tips.js`

**目标**：为 tips <50 字的食谱补充选材技巧、火候要点、保存方法

#### 2.2.1 数据源设计

创建 tips 模板库 `data/tips-templates.json`，按烹饪方法（蒸、煮、炒、炸、烤、炖、拌）组织：

```json

    "troubleshoot": ["如果口感偏老，下次缩短{time}分钟试试"]
  },
  "炒": {
    "selection": ["{主食材}选颜色鲜亮、质地紧实的，手感重的水分足"],
    "heat": ["热锅凉油，大火快炒不超过{time}分钟，锁住水分和脆嫩"],
```

#### 2.2.2 增强逻辑

```

  2. 从模板库选取 1 条选材技巧 + 1 条火候要点 + 1 条保存方法
  3. 用 \\n 分隔追加到原 tips 末尾

```

### 2.4 P3：食材分组（数据模型问题—已知限制）

**现状**：当前 `ingredients` 字段为扁平 JSON 数组 `[{name, amount, unit}]`，无分组结构。

**影响**：
- 前端无法按"主料/辅料/调料/腌料"分类展示
- 用户难以快速区分核心食材和辅助调料
- 购物清单无法按品类分组

**标记为已知限制**：本迭代不解决数据模型问题，原因：
1. 需要修改 Sequelize model 的 ingredients 字段结构（从扁平数组改为分组对象）
2. 涉及数据库迁移、seed 重构、前端解析逻辑全面改动
3. 工作量超出"内容增强"迭代范围，建议列入 P4（数据模型优化迭代）

**临时方案**：前端可基于 ingredients 中的 name/amount 做简单推断分组（如"克数大的可能是主料"），但不作为本迭代交付项。

## 3. 前端展示优化方案

### 3.1 Story 区块视觉增强

**现状**：折叠式手风琴，展开后为灰色背景纯文本

**目标**：打造沉浸式阅读体验

#### 3.1.1 引用样式（blockquote 风格）

```
┌────────────────────────────────────────┐
│  📖 食谱故事                            │
│  ┌──────────────────────────────────┐  │
│  │  "哈尔滨冬天，朋友家大铁锅里      │  │
│  │   炖着排骨豆角土豆玉米南瓜        │  │
│  │   咕嘟冒泡..."                   │  │
│  │                                  │  │
│  │              —— 东北乱炖的故事    │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

**CSS 实现**：
- 左侧竖线装饰（border-left: 3px solid primary color）
- 斜体排版（font-style: italic）
- 大号引号装饰（::before "“" ::after "”" 大号半透明文字）
- 圆角卡片背景（暖色/微黄底色，模拟纸张质感）

#### 3.1.2 背景装饰

- 可选的微纹理背景（CSS pattern: 细线纹或圆点图案）

### 3.4 描述区域排版优化

**现状**：`<p className="detail-desc">` 纯文本，15px/1.75 行高

**目标**：提升信息密度和阅读舒适度

#### 3.4.1 排版增强

- **增大字号**：15px → 16px（匹配现代阅读标准）
- **优化行高**：1.75 → 1.8（更松弛舒适的阅读节奏）
- **最大宽度限制**：max-width: 680px（避免过长行导致的阅读疲劳）
- **首字下沉**（可选）：对长描述（>120字）的首字做 artistic 处理
- **段落间距**：margin-bottom 24px → 32px

#### 3.4.2 视觉分隔

- 描述与标题之间增加一条细装饰线（渐变线 or 虚线）
- 描述末尾增加一个微小的装饰元素（如 · · ·）

### 3.5 暗色模式适配

**现状**：已有部分 `[data-theme='dark']` 覆盖，但不完整
