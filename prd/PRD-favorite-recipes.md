## 1. 功能概述

| 项目 | 内容 |
|------|------|
| 功能名称 | 用户收藏食谱 |
| 所属产品 | 美食食谱网站（food-website） |
| 核心价值 | 让用户保存喜欢的食谱以便后续查阅，提升回访率与用户体验 |
| 用户故事 | 作为登录用户，我在浏览食谱时可以将喜欢的食谱收藏，之后可以随时在我的收藏列表中快速找到它们，无需再次搜索 |

---

## 2. 功能描述

### 2.1 核心功能点

1. **收藏按钮**：在食谱详情页，显示收藏按钮（未收藏时为空心/轮廓图标，已收藏时为实心/填充图标）。
2. **状态切换**：点击收藏按钮后，按钮状态实时切换，无需刷新页面。
3. **收藏列表**：用户提供"我的收藏"页面，展示所有已收藏的食谱。
4. **取消收藏**：在食谱详情页或收藏列表中，用户可再次点击按钮取消收藏。

### 2.2 交互规则

- **未登录用户**：点击收藏按钮时，提示登录/引导去登录页面，不记录收藏。
- **已登录用户**：收藏操作即时生效，UI 实时反馈。
- **重复收藏**：同一食谱不可重复收藏，后端应对重复请求返回幂等响应（返回当前状态，不报错）。
- **收藏计数**（可选）：食谱详情页可显示该食谱被收藏的总次数。

## 3. 页面流程

### 3.1 食谱详情页（Recipe Detail Page）

```
用户进入食谱详情页
       ↓
  是否已登录？
   ↙         ↘
  否            是
   ↓              ↓
引导登录    检查当前食谱是否已在收藏列表
   ↓              ↓
结束       ↙          ↘
        是            否
         ↓              ↓
   显示"已收藏"    显示"收藏"
         ↓              ↓
    点击→取消收藏    点击→添加收藏
         ↓              ↓
   显示"收藏"      显示"已收藏"
```

### 3.2 我的收藏页（My Favorites Page）

```
用户进入"我的收藏"页
       ↓
  请求后端收藏列表 API
       ↓
  是否有收藏记录？
   ↙         ↘
  无            有
   ↓              ↓
显示空状态页面    展示收藏食谱列表
（提示语+引导）   （卡片/列表形式）
                        ↓
                  点击任一食谱 → 进入详情页
                        ↓
                  点击取消收藏 → 从列表移除（局部更新）
```

## 4. 数据模型

### 4.1 Favorite 收藏记录

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `id` | string（UUID） | ✅ | 收藏记录唯一标识符 |
| `userId` | string（UUID） | ✅ | 收藏者用户 ID |
| `recipeId` | string（UUID） | ✅ | 被收藏的食谱 ID |
| `createdAt` | ISO 8601 DateTime | ✅ | 收藏时间，格式：`YYYY-MM-DDTHH:mm:ss.sssZ` |

### 4.2 索引与约束

- **唯一索引**：`UNIQUE(userId, recipeId)` — 防止同一用户重复收藏同一食谱
- **查询索引**：`INDEX(userId, createdAt DESC)` — 按用户查询收藏列表，按时间倒序

### 4.3 相关实体（参考）

```
User {
  id: string
  username: string
  email: string
  ...
}

Recipe {
  id: string
  title: string
  coverImage: string
  authorId: string
  ...
}
```

> 备注：Favorite 表通过 `userId` 关联 User，`recipeId` 关联 Recipe，查询时建议 JOIN Recipe 表以获取食谱基本信息（标题、封面、作者等）。

## 5. API 需求

### 5.1 添加收藏

```
POST /api/favorites
```

**请求头**：
- `Authorization: Bearer <token>`（必需）

**请求体**：
```json
{
  "recipeId": "string (UUID)"
}
```

**成功响应**（201 Created）：
```json
{
  "code": 0,
  "message": "收藏成功",
  "data": {
    "id": "uuid",
    "userId": "uuid",
    "recipeId": "uuid",
    "createdAt": "2026-05-13T15:32:00.000Z"
  }
}
```

**重复收藏响应**（200 OK，幂等）：
```json
{
  "code": 0,
  "message": "已收藏",
  "data": null
}
```

**错误响应**（401 Unauthorized）：
```json
{
  "code": 401,
  "message": "请先登录",
  "data": null
}
```

---

### 5.2 取消收藏

```
DELETE /api/favorites/{recipeId}
```

**请求头**：
- `Authorization: Bearer <token>`（必需）

**路径参数**：
- `recipeId`：食谱 ID

**成功响应**（200 OK）：
```json
{
  "code": 0,
  "message": "取消收藏成功",
  "data": null
}
```

**未收藏时响应**（200 OK，幂等）：
```json
{
  "code": 0,
  "message": "未收藏，无需取消",
  "data": null
}
```

---

### 5.3 获取收藏列表

```
GET /api/favorites?page=1&pageSize=20
```

**请求头**：
- `Authorization: Bearer <token>`（必需）

**查询参数**：
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `page` | integer | 1 | 页码（从 1 开始） |
| `pageSize` | integer | 20 | 每页条数（最大 100） |

**成功响应**（200 OK）：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 42,
    "page": 1,
    "pageSize": 20,
    "list": [
      {
        "id": "uuid",
        "userId": "uuid",
        "recipeId": "uuid",
        "createdAt": "2026-05-10T08:00:00.000Z",
        "recipe": {
          "id": "uuid",
          "title": "红烧肉",
          "coverImage": "https://example.com/images/hongshaorou.jpg",
          "author": "张三",
          "cookTime": 60
        }
      }
    ]
  }
}
```

---

### 5.4 查询单条收藏状态（可选，用于详情页初始化）

```
GET /api/favorites/{recipeId}/status
```

**成功响应**（200 OK）：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "isFavorited": true,
    "favoriteId": "uuid" // 如已收藏则返回记录 ID
  }
}
```

## 6. 验收标准

### 6.1 功能验收

| ID | 验收条件 | 测试方法 |
|----|----------|----------|
| AC-01 | 未登录用户点击收藏按钮，系统提示登录并不记录收藏 | 游客状态点击，验证提示出现 |
| AC-02 | 已登录用户点击"收藏"按钮，收藏记录入库，UI 立即变为"已收藏" | 点击后刷新页面，状态仍为已收藏 |
| AC-03 | 已收藏状态下点击按钮，收藏记录删除，UI 立即恢复为"收藏" | 点击后刷新页面，状态已恢复 |
| AC-04 | "我的收藏"页正确展示用户所有收藏的食谱（包含基本信息） | 收藏 3+ 食谱，访问收藏页验证 |
| AC-05 | 收藏列表中点击食谱可正常跳转至详情页 | 点击卡片，验证 URL 和页面内容 |
| AC-06 | 收藏列表支持分页，page/pageSize 参数生效 | 收藏 >20 条，翻页验证数据正确 |
| AC-07 | 同一食谱重复收藏不产生多条记录 | 重复点击添加收藏，数据库仅 1 条 |
| AC-08 | API 错误时前端有容错处理（网络异常提示） | 断网测试，验证友好提示 |

### 6.2 非功能性验收

| ID | 验收条件 |
|----|----------|
| NF-01 | 收藏/取消收藏操作响应时间 ≤ 500ms（网络正常时） |
| NF-02 | 收藏列表首次加载时间 ≤ 1.5s（含封面图加载） |
| NF-03 | 移动端（375px宽度）收藏按钮可正常点击，图标清晰 |
| NF-04 | 收藏功能不污染其他页面的数据状态 |
